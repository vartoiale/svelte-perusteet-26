# Teoria 2: javascript

## Javascript on selainten tukema ohjelmointikieli

Javascript on selaimessa toimivana ohjelmointikielinä pääasiallinen web-kehittäjien käyttämä kieli.
Se on lisäksi ainoa suoraan selaimien tukema kieli tänä päivänä.
Tämän takia javascriptin osaaminen on web-kehittäjälle tärkeää.

Javascript ei ole kuitenkaan ainoa ohjelmointikieli, jota web-kehityksessä käytetään.
Palvelimella voidaan käyttää mitä tahansa ohjelmointikieltä.
Selaimessakin voidaan niin kutsutun webassemblyn avulla käyttää käännetyssä muodossa mitä tahansa sellaista ohjelmointikieltä, jolle löytyy webassemblyä tukeva kääntäjä.

Tässä kuitenkin siis lähdemme tutustumaan Javascriptiin ohjelmointikielenämme.

## Aloitamme javascriptiin siirtymisen svelte-sovelluskehyksen avulla

Aloitamme tämän lähestymisen kevyesti, sukeltamalla ensin syvään päätyyn, 
ja komponenttikirjastoihin. 
Ihan vain nähdäksemme, että matalimmillaan syvä pääty ei oikeastaan ole vielä kovinkaan syvä.

Opetussuunnitelman mukaisesti komponenttikirjastot tulevat virallisesti vasta paljon myöhäisemmässä vaiheessa.
Näin alkuun kuitenkin pulahdamme vain kevyesti Svelte-nimisen komponenttikirjaston parissa.

Sveltestä voit lukea lisää tiedostosta: [`02-01-js-ja-svelte.md`](02-01-js-ja-svelte.md).

Svelte dokumentaation rinnalla haluat myös lukea alta löytyvää javascriptin perusteiden ohjeistusta.

## javascriptin perusteet

Lähdemme siis tutustumaan javascriptin perusteisiin.

Tämä tapahtuu alkuun käyttämällä selaimen kehittäjätyökaluista löytyvää interaktiivista konsolia.
Se on vähän samanlainen työkalu kuin windowsista ja linuxista löytyvä komentorivikehote.
Siis esimerkiksi windowsin powershell-työkalu.

Kuitenkin, siinä missä komentorivikehotteella käytetään sovelluksia,
selaimen kehittäjätyökaluista löytyvään konsoliin kirjoitetaankin oikeaa javascriptiä.

Javascriptistä voit lukea käytäntö-osuudesta tiedostosta: [`/materiaali/kaytanto/02-javascript/javascript-repl.md`](/materiaali/kaytanto/02-javascript/javascript-repl.md).
